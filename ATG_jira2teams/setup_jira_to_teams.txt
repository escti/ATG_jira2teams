# Passo a Passo para Configurar o Script Jira to Teams no Oracle Linux 8 ARM

## 1. Atualizar o Sistema
Certifique-se de que o sistema está atualizado para evitar problemas de dependências.
```bash
sudo dnf update -y
```

## 2. Instalar Python 3.9 (Versão mais recente no Oracle Linux 8)
O Oracle Linux 8 oferece Python 3.9 via AppStream Modules, compatível com ARM.
```bash
sudo dnf module install -y python39
sudo dnf install -y python39-pip python39-devel
```
Verifique a instalação:
```bash
python3.9 --version
pip3.9 --version
```

## 3. Instalar Dependências do Script
Instale as bibliotecas Python necessárias (requests, pytz).
```bash
pip3.9 install requests pytz
```

## 5. Verificar Conectividade
Confirme que a máquina tem acesso ao Jira e ao Microsoft Teams:
```bash
ping jira.atlassian.net
curl -I https://outlook.office.com
```

## 6. Criar Diretório e Configurar o Script
Crie o diretório para o script e configure permissões:
```bash
chmod +x /home/atg/scripts/jira_to_teams.py
```
Salve o script fornecido (jira_to_teams.py) em /home/atg/scripts/jira_to_teams.py.

## 7. Testar o Script Manualmente
Execute o script para verificar se está funcionando:
```bash
python3.9 /home/atg/scripts/jira_to_teams.py
```
- Verifique o log em /home/atg/scripts/jira_to_teams.log para mensagens de sucesso ou erros.
- Confirme no Microsoft Teams se os dois cards foram recebidos corretamente.

## 8. Agendar o Script com Cron (Segunda a Sexta, 7:59 às 17:59)
Configure o cron para executar o script a cada hora, das 7:59 às 17:59, de segunda a sexta, carregando as variáveis de ambiente do ~/.bashrc.
Abra o crontab:
```bash
crontab -e
```
Adicione a linha abaixo:
```bash
59 7-17 * * 1-5 /bin/bash -c 'source /home/atg/scripts/jira2teams.env && /usr/bin/python3.9 /home/atg/scripts/jira_to_teams.py >> /home/atg/scripts/cron.log 2>&1'
```
Essa linha carrega o ~/.bashrc antes de executar o script, garantindo que as variáveis de ambiente (JIRA_SERVER, JIRA_USERNAME, etc.) estejam disponíveis.
Verifique a configuração do cron:
```bash
crontab -l
```

## 9. Verificar Logs
- Log do script: /home/atg/scripts/jira_to_teams.log
- Log do cron: /home/atg/scripts/cron.log
- Log do sistema (se necessário): sudo tail -f /var/log/cron

## 10. Depuração
- **Erro de Python**: Verifique as bibliotecas instaladas:
  ```bash
  pip3.9 list
  ```
- **Erro de Variáveis de Ambiente no Cron**: Teste o ambiente do cron manualmente:
  ```bash
  /bin/bash -c 'source /home/atg/.bashrc && env | grep -E "JIRA|TEAMS"'
  ```
  Isso deve mostrar as variáveis JIRA_SERVER, JIRA_USERNAME, JIRA_PASSWORD e TEAMS_WEBHOOK_URL. Se não aparecerem, verifique o ~/.bashrc.
- **Erro de Conexão ou API**: Teste a autenticação com o Jira:
  ```bash
  curl -u $JIRA_USERNAME:$JIRA_PASSWORD -X GET "$JIRA_SERVER/rest/api/3/myself"
  ```
  Teste a query JQL manualmente:
  ```bash
  curl -u $JIRA_USERNAME:$JIRA_PASSWORD -X POST -H "Content-Type: application/json" \
  --data '{"jql":"assignee = currentUser() AND statusCategory != Done","maxResults":10,"fields":["key","summary","updated"]}' \
  $JIRA_SERVER/rest/api/3/search
  ```
- **Erro de Cron**: Confirme que o serviço crond está ativo:
  ```bash
  sudo systemctl status crond
  sudo systemctl enable crond
  sudo systemctl start crond
  ```
- **Campos Personalizados**: As queries JQL foram validadas na UI do Jira. Se houver erros, confirme os nomes dos campos em Settings > Issues > Custom fields. Se forem custom fields (ex: customfield_12345), substitua no script. Exemplo:
  ```python
  "jql": 'assignee IS EMPTY AND customfield_12345 = "DC - Banco de Dados (DBA)" AND statusCategory != Done AND ("Tempo de Primeira Resposta" = breached() OR "Tempo de Primeira Resposta" <= remaining("1h"))'
  ```

## 11. Notas Adicionais
- O script usa a API /rest/api/3/search/jql com fallback para /rest/api/3/search para evitar erros de endpoint obsoleta.
- Se precisar de Python mais recente (ex: 3.11), compile manualmente:
  ```bash
  sudo dnf install -y gcc openssl-devel bzip2-devel libffi-devel zlib-devel
  wget https://www.python.org/ftp/python/3.11.9/Python-3.11.9.tar.xz
  tar -xf Python-3.11.9.tar.xz
  cd Python-3.11.9
  ./configure --enable-optimizations
  make -j$(nproc)
  sudo make altinstall
  ```
  Atualize o script e o cron para usar python3.11 (ex: substitua python3.9 por python3.11 no crontab).
- Para maior segurança, considere usar um gerenciador de segredos (ex: HashiCorp Vault) em vez de ~/.bashrc para credenciais.
